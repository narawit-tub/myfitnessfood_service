# Project Overview Template

## Basic Information

- **Project Name:** my fitness food
- **Project Type:** backend service for mobile app

## Project Description

### Problem Statement

To build good fitness help, exercise need to come with high quality food with enough of macro nutrience and calories to build the strong fitness health. It's too challenge for beginner to stay on track. Start from how they know how many calories or macro nutrience left for them to find more food to fill in, to what food or action plan they can do to complete the daily target goal.

### Solution Overview

Solution are

1. Help user track their meal calories or macro nutrience, if they do not know. Using AI to find the average.
2. Suggesting user what they can do to achive the daily goal, like what food they can eat to complete the diet.
3. follow from solution 2, if action is about making own dish, what ingredience or steps to cook

### Target Users/Audience

1. Everyone who want good fitness health

### Key Objectives

1. Help user stay on track of food they consume.
2. Reduce the efforts for user to find the good food resource to achive their goal

## Technical Details

### Technology Stack

- Runtime: Node.js 24+ or Bun
- Database: PostgreSQL (on Supabase or Neon for serverless)
- ORM: Prisma 7 (Serverless-ready)
- API Layer: NestJS + Fastify
- Validation: Zod
- Auth: Clerk (Managed) or @nestjs/jwt (Custom)
- Docs: Scalar (OpenAPI)
- Deployment: AWS Lambda (via AWS Lambda Web Adapter) or Fargate
